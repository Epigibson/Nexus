## Paso 1: Instalar el CLI

<Tabs>
  <Tab title="Windows (PowerShell)">
    ```powershell
    # Descargar y compilar
    git clone https://github.com/Epigibson/antigravity.git
    cd antigravity/core
    go build -o antigravity.exe ./cmd/antigravity

    # Instalar globalmente
    $installDir = "$env:USERPROFILE\.antigravity\bin"
    New-Item -ItemType Directory -Force -Path $installDir | Out-Null
    Copy-Item antigravity.exe "$installDir\antigravity.exe" -Force

    # Agregar al PATH (permanente, solo la primera vez)
    $currentPath = [System.Environment]::GetEnvironmentVariable("Path", "User")
    if ($currentPath -notlike "*\.antigravity\bin*") {
        [System.Environment]::SetEnvironmentVariable("Path", "$currentPath;$installDir", "User")
    }
    ```

    <Warning>
      Reinicia tu terminal después de agregar al PATH para que tome efecto.
    </Warning>
  </Tab>
  <Tab title="macOS / Linux">
    ```bash
    # Descargar y compilar
    git clone https://github.com/Epigibson/antigravity.git
    cd antigravity/core
    go build -o antigravity ./cmd/antigravity

    # Instalar globalmente
    mkdir -p ~/.antigravity/bin
    cp antigravity ~/.antigravity/bin/
    chmod +x ~/.antigravity/bin/antigravity

    # Agregar al PATH (añadir a ~/.bashrc o ~/.zshrc)
    echo 'export PATH="$HOME/.antigravity/bin:$PATH"' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>
</Tabs>

Verifica la instalación:

```bash
antigravity version
# → Antigravity v0.1.0
```

<Check>
  Si ves `Antigravity v0.1.0`, la instalación fue exitosa.
</Check>

## Paso 2: Crear cuenta y API Key

<Steps>
  <Step title="Registrarte en el Dashboard">
    Ve a [antigravity-production-a677.up.railway.app](https://antigravity-production-a677.up.railway.app) y crea tu cuenta.
  </Step>
  <Step title="Crear un proyecto">
    En el dashboard, crea un proyecto y agrega entornos (development, staging, production). Configura los CLI profiles (GitHub, AWS, etc.) y variables de entorno (API keys, secrets).
  </Step>
  <Step title="Generar una API Key">
    Ve a **Configuración → API Keys → Generar Nueva Key**.

    <Warning>
      La key completa (`ag_live_...`) solo se muestra **una vez**. Cópiala antes de cerrar el modal.
    </Warning>
  </Step>
</Steps>

## Comandos disponibles

| Comando | Descripción |
|---------|-------------|
| `antigravity login` | Autenticarte con tu API key |
| `antigravity sync` | Sincronizar proyectos del cloud |
| `antigravity status` | Ver estado de conexión |
| `antigravity switch <proyecto> --env <env>` | Cambiar contexto completo |
| `antigravity list` | Listar proyectos locales |
| `antigravity init` | Crear antigravity.yaml local |
| `antigravity logout` | Desconectar del cloud |
| `antigravity version` | Ver versión |

## Troubleshooting

<AccordionGroup>
  <Accordion title="'antigravity' no se reconoce como comando">
    Asegúrate de que `~/.antigravity/bin` está en tu PATH y que reiniciaste la terminal después de instalarlo.

    **Windows**: Verifica con `$env:Path -split ";" | Where-Object { $_ -like "*antigravity*" }`

    **macOS/Linux**: Verifica con `echo $PATH | tr ':' '\n' | grep antigravity`
  </Accordion>
  <Accordion title="Authentication failed al hacer login">
    - Verifica que copiaste la API key completa (empieza con `ag_live_`)
    - Genera una nueva key desde el dashboard si la anterior expiró
    - Verifica conexión a internet
  </Accordion>
  <Accordion title="'go' no se reconoce (conflicto con alias)">
    Si tienes un alias de PowerShell que reemplaza `go` (por ejemplo, para `git checkout`):

    ```powershell
    # Renombra tu alias en $PROFILE
    # Cambia: function go { ... }
    # A:      function ggo { ... }
    ```

    O usa el path completo: `& "C:\Program Files\Go\bin\go.exe" build ...`
  </Accordion>
  <Accordion title="Error 500 al generar API Key">
    Si ves un error interno al crear la key, espera 2 minutos para que Railway redeployee tras un push reciente.
  </Accordion>
</AccordionGroup>
